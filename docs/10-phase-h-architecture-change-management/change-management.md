# Architecture Change Management

# Propósito

Gestionar cambios en la arquitectura objetivo sin perder control 
ni bloquear evolución del negocio.

# Tipos de cambio

| Tipo | Ejemplo | Tratamiento |
|---|---|---|
| Menor | Ajuste de estándar, actualización de template | Revisión del equipo de arquitectura |
| Medio | Nueva integración, nuevo servicio compartido | Revisión Architecture Board |
| Mayor | Nuevo core, nuevo proveedor estratégico, cambio de plataforma | Business case y aprobación ejecutiva |
| Emergencia | Cambio por incidente o vulnerabilidad crítica | Aprobación rápida y revisión posterior |

# Backlog arquitectónico

Debe contener:

- deuda técnica priorizada;
- excepciones abiertas;
- estándares pendientes;
- gaps de plataforma;
- riesgos tecnológicos;
- iniciativas de modernización.

# Cadencia

| Ceremonia | Frecuencia | Participantes |
|---|---|---|
| Architecture Board | Quincenal | Arquitectura, seguridad, data, plataforma |
| Technology Radar | Trimestral | Tecnología, seguridad, squads |
| Portfolio Review | Mensual | Negocio, PMO, tecnología |
| Exception Review | Mensual | Architecture Board |
