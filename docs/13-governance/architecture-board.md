# Architecture Board

# Misión

Aprobar, priorizar y gobernar decisiones arquitectónicas que impactan múltiples dominios, riesgo operativo, seguridad, inversión o estrategia tecnológica.

# Composición

- Head/Lead de Arquitectura.
- Arquitectura de soluciones.
- Arquitectura de seguridad.
- Arquitectura de datos.
- Plataforma/Infraestructura.
- Operaciones/SRE.
- Representante de producto o negocio según iniciativa.
- Compliance cuando aplique.

# Decisiones que requieren board

- Nuevo proveedor estratégico.
- Nueva plataforma tecnológica.
- Excepción a estándar de seguridad.
- Exposición de API externa crítica.
- Integración punto a punto no estándar.
- Uso de datos sensibles en nuevos modelos.
- Cambio relevante en arquitectura target.
- Endeudamiento técnico de alto impacto.

# Resultado esperado

Cada sesión debe producir al menos uno:

- decisión aprobada;
- decisión rechazada;
- requerimiento adicional;
- excepción temporal;
- ADR solicitado;
- riesgo escalado.
